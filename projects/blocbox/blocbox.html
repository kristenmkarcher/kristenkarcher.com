    What's Blocbox?

    Blocbox is a tool designed for collecting and storing information. You can create simple notes, store images, as well as add and share links.

    What is Blocbox trying to solve?

    Blocbox came to me with an outdated website relying on dated design concepts. My role was to come up with new concepts, user flows, layouts and bringing a new modern look and feel to their application.

    I developed a new brand identity for the site, including a redesigned logo, fresh page layout and a cleaner overall solution for user interaction. The link below is the newly created dashboard for Blocbox.

    User Surveys

    User testing was useful in that it helped to determine if the new design provided clarity to the user versus the old design, and which design users preferred. It also measured the users competency through success rate of completing certain tasks, and the amount of time to complete those tasks.

    Through email and social media, my survey was completed by 34 different respondents. Below are my findings.

    100% of users prefer to browse online using their smart phones


    20% of users save links or images they find interesting online


    69% of users bookmark links or images they find interesting online


    82% of users bookmark sites using their browser


    48% of users write notes to remember information for later


    57% of users use a smart phone app to write and keep notes


    58% of users share resources and collaborate with their co-workers

    User flow using Balsamiq

    I created a user flow based on both new and returning users. For new users, a flow was created to explore the features of Blocbox, sign up using social logins or email and password and navigating to the dashboard after signing up. For returning users, a flow was created to sign in using social logins or email and password and the ability to reset their password. A flow for shared tasks was also created for both new and returning users which included adding a link, image or note, creating a new box, updating account settings, inviting and managing collaborators and viewing content.

    High-fidelity Wireframes

    I used wireframes to create sketches to display content placement. They were a good starting point as elements were added to the page to determine the best structure, and enable focus on navigation so users understand important actions.

    Designing the Dashboard

    I wanted to have the dashboard digitally represent a number of popular items currently tied to the users account. Users are able to see their most active items, or items they have favorited that were added by other users.

    Designing the Account Page

    The account page is simple and easy to navigate. At first glance, the user can immediately see all relevant information regarding their account.

    Designing the Home Page

    The user is first introduced to an example of the dashboard, before being introduced to pricing plans. Through this, the user can preview what the application offers prior to viewing any pricing information. Reviews are also highlighted in order for the visitor to see what other users are saying about the application.

    Preference Test for High-fidelity Wireframes

    Through wireframe testing, final options came down to how easy the layout would be to navigate. Ultimately, option A won out, because of the more natural and appealing layout of the elements on each row.

    Preference Test for Final Design

    After creating a preference test for the final design, it was decided through extensive user testing option B would ultimately be the choice.

    What I learned Throughout the Process

    The user feedback I got from the survey was very helpful in determining the layout and usability of the site. The biggest challenge of the project was filtering through the feedback I got and taking the results to determine the best possible layout for the site. This project helped me to learn how to take user feedback and apply it to my design and learn how to use various tools and the areas where they should be implemented. I will use the entire process learned during the project in the future from idea creation, to design, to deployment.
